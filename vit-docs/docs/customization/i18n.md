# Internationalization (i18n)

Vit Design System includes built-in internationalization support through the `VitStrings` class. This allows you to customize all text strings used by components for different languages and locales.

## Overview

The `VitStrings` class contains all text strings used internally by Vit components, including:
- Form field labels and hints
- Validation error messages
- Common UI text (search, confirm, etc.)
- Gender and language options
- Date and country labels

## Basic Usage

### Default Strings (English)

By default, all components use English strings:

```dart
VitApp(
  theme: VitTheme(),
  home: MyHomePage(),
)
```

### Custom Strings

Override specific strings to match your needs:

```dart
VitApp(
  theme: VitTheme(
    bitStrings: VitStrings(
      confirm: 'OK',
      email: 'Email Address',
      password: 'Your Password',
      required: 'This field is mandatory',
    ),
  ),
  home: MyHomePage(),
)
```

## Available Strings

### Common Strings

General-purpose strings used across components:

```dart
VitStrings(
  confirm: 'Confirm',
  searchHint: 'Search...',
  noResultsFound: 'No results found',
  required: 'This field is required',
  search: 'Search',
)
```

| Property | Default | Usage |
|----------|---------|-------|
| `confirm` | `'Confirm'` | Confirmation buttons |
| `searchHint` | `'Search...'` | Search input placeholder |
| `noResultsFound` | `'No results found'` | Empty search/select results |
| `required` | `'This field is required'` | Required field validation |
| `search` | `'Search'` | Search field label |

## Dynamic Language Switching

### Using with Flutter Localization

Integrate with Flutter's localization system:

```dart
import 'package:flutter_localizations/flutter_localizations.dart';

class AppLocalizations {
  final Locale locale;

  AppLocalizations(this.locale);

  static VitStrings getStrings(Locale locale) {
    switch (locale.languageCode) {
      case 'es':
        return VitStrings(
          confirm: 'Confirmar',
          email: 'Correo electrónico',
        );
      case 'pt':
        return VitStrings(
          confirm: 'Confirmar',
          email: 'E-mail',
        );
      case 'fr':
        return VitStrings(
          confirm: 'Confirmer',
          email: 'E-mail',
        );
      default:
        return VitStrings();
    }
  }
}

class MyApp extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return VitApp(
      localizationsDelegates: [
        GlobalMaterialLocalizations.delegate,
        GlobalWidgetsLocalizations.delegate,
        GlobalCupertinoLocalizations.delegate,
      ],
      supportedLocales: [
        Locale('en', 'US'),
        Locale('es', 'ES'),
        Locale('pt', 'BR'),
        Locale('fr', 'FR'),
      ],
      onGenerateTitle: (context) {
        final locale = Localizations.localeOf(context);
        return 'My App';
      },
      theme: VitTheme(
        bitStrings: AppLocalizations.getStrings(Locale('en')),
      ),
      home: MyHomePage(),
    );
  }
}
```

### Runtime Language Change

Change language dynamically:

```dart
class LanguageSwitcher extends StatefulWidget {
  @override
  _LanguageSwitcherState createState() => _LanguageSwitcherState();
}

class _LanguageSwitcherState extends State<LanguageSwitcher> {
  Locale _currentLocale = Locale('en', 'US');

  void _changeLanguage(String languageCode) {
    setState(() {
      _currentLocale = Locale(languageCode);
    });
  }

  @override
  Widget build(BuildContext context) {
    return VitApp(
      locale: _currentLocale,
      theme: VitTheme(
        bitStrings: AppLocalizations.getStrings(_currentLocale),
      ),
      home: Scaffold(
        body: Column(
          children: [
            VitButton(
              text: 'English',
              onPressed: () => _changeLanguage('en'),
            ),
            VitButton(
              text: 'Español',
              onPressed: () => _changeLanguage('es'),
            ),
            VitButton(
              text: 'Português',
              onPressed: () => _changeLanguage('pt'),
            ),
          ],
        ),
      ),
    );
  }
}
```

## Best Practices

### Completeness

When providing custom strings, override all strings that your app uses to maintain consistency:

```dart
VitStrings(
  confirm: 'Confirmar',
  email: 'Correo electrónico',
  emailHint: 'Ingrese su correo',
  emailError: 'Correo inválido',
  password: 'Contraseña',
  passwordHint: 'Ingrese su contraseña',
)
```

### Consistency

- Use the same terminology throughout your app
- Keep translation style consistent (formal vs informal)
- Maintain similar message lengths to avoid layout issues

### Context

- Consider cultural context when translating
- Adapt phrases to feel natural in each language
- Some phrases may need restructuring, not just translation

### Testing

- Test all supported languages in your app
- Check for text overflow with longer translations
- Verify right-to-left languages if supported

## Complete i18n Example

Here's a complete example with proper localization structure:

```dart
import 'package:flutter/material.dart';
import 'package:flutter_localizations/flutter_localizations.dart';
import 'package:vit_design_system/vit_design_system.dart';

class AppStrings {
  static VitStrings english() => VitStrings(
    confirm: 'Confirm',
    email: 'Email',
    emailHint: 'Enter your email',
    password: 'Password',
    passwordHint: 'Enter your password',
    search: 'Search',
    required: 'This field is required',
  );

  static VitStrings spanish() => VitStrings(
    confirm: 'Confirmar',
    email: 'Correo electrónico',
    emailHint: 'Ingrese su correo',
    password: 'Contraseña',
    passwordHint: 'Ingrese su contraseña',
    search: 'Buscar',
    required: 'Este campo es obligatorio',
  );

  static VitStrings portuguese() => VitStrings(
    confirm: 'Confirmar',
    email: 'E-mail',
    emailHint: 'Digite seu e-mail',
    password: 'Senha',
    passwordHint: 'Digite sua senha',
    search: 'Pesquisar',
    required: 'Este campo é obrigatório',
  );

  static VitStrings forLocale(Locale locale) {
    switch (locale.languageCode) {
      case 'es':
        return spanish();
      case 'pt':
        return portuguese();
      default:
        return english();
    }
  }
}

class MyApp extends StatefulWidget {
  @override
  _MyAppState createState() => _MyAppState();
}

class _MyAppState extends State<MyApp> {
  Locale _locale = Locale('en');

  void _setLocale(Locale locale) {
    setState(() {
      _locale = locale;
    });
  }

  @override
  Widget build(BuildContext context) {
    return VitApp(
      locale: _locale,
      supportedLocales: [
        Locale('en'),
        Locale('es'),
        Locale('pt'),
      ],
      localizationsDelegates: [
        GlobalMaterialLocalizations.delegate,
        GlobalWidgetsLocalizations.delegate,
        GlobalCupertinoLocalizations.delegate,
      ],
      theme: VitTheme(
        bitStrings: AppStrings.forLocale(_locale),
      ),
      home: MyHomePage(onLocaleChange: _setLocale),
    );
  }
}
```
