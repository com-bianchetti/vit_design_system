# Card

Container components for grouping related content.

Cards are versatile containers that provide consistent styling and structure for displaying related information. The VitCard component offers extensive customization options including elevation, borders, custom colors, gradients, and interactive behaviors. Use cards to organize content into logical sections, create visual hierarchy, and provide clickable areas for navigation.

<FlutterPreview story="VitCard" />

## Import

```dart
import 'package:vit_design_system/vit_design_system.dart';
```

## Basic Usage

### Simple Card

The simplest form of a card wraps any widget child with consistent styling. Cards automatically apply theme-based colors, border radius, and padding based on your design system configuration.

```dart
VitCard(
  child: VitText('Card content'),
)
```

### Card with Variants

VitCard provides four distinct visual variants that use different color schemes from your theme. These variants help establish visual hierarchy and distinguish between different types of content in your application.

**Standard Variant** - Uses the default card background color from your theme:

```dart
VitCard(
  variant: VitCardVariant.standard,
  child: VitText('Standard card'),
)
```

**Elevated Variant** - Uses a slightly elevated appearance with different background color:

```dart
VitCard(
  variant: VitCardVariant.elevated,
  child: VitText('Elevated variant'),
)
```

**Variant Style** - Uses an alternative color scheme for visual distinction:

```dart
VitCard(
  variant: VitCardVariant.variant,
  child: VitText('Variant style'),
)
```

**Elevated Variant Style** - Combines elevated styling with variant colors:

```dart
VitCard(
  variant: VitCardVariant.elevatedVariant,
  child: VitText('Elevated variant style'),
)
```

### Card with Elevation

Add depth and shadow to cards using the elevation property. Higher elevation values create stronger shadows, making the card appear to float above the surface. This is useful for emphasizing important content or creating layered interfaces.

```dart
VitCard(
  elevation: 4,
  child: Column(
    children: [
      VitText('Card with shadow'),
    ],
  ),
)
```

## Customization Features

### Custom Colors and Gradients

Customize the visual appearance with custom background colors or gradients. When using gradients, the card creates stunning visual effects perfect for hero sections or featured content.

```dart
VitCard(
  backgroundColor: Colors.blue.shade50,
  child: VitText('Custom background color'),
)
```

```dart
VitCard(
  gradient: LinearGradient(
    colors: [Colors.purple, Colors.blue],
    begin: Alignment.topLeft,
    end: Alignment.bottomRight,
  ),
  child: VitText('Gradient card', style: TextStyle(color: Colors.white)),
)
```

### Borders and Border Radius

Control the card's border appearance with border visibility, color, width, and corner radius. Borders can help define card boundaries, especially on backgrounds with similar colors.

```dart
VitCard(
  showBorder: true,
  borderColor: Colors.grey.shade300,
  borderWidth: 2,
  borderRadius: BorderRadius.circular(16),
  child: VitText('Card with custom border'),
)
```

### Padding and Dimensions

Control the card's internal spacing and overall size. Use padding to adjust content spacing, and set explicit dimensions when you need fixed-size cards like in grid layouts.

```dart
VitCard(
  padding: EdgeInsets.all(24),
  width: 200,
  height: 150,
  child: VitText('Fixed size card'),
)
```

### Visual Density

Adjust card padding automatically based on visual density. This ensures consistent spacing across your app that adapts to different screen sizes or user preferences.

```dart
VitCard(
  visualDensity: VisualDensity.compact,
  child: VitText('Compact padding'),
)
```

```dart
VitCard(
  visualDensity: VisualDensity.comfortable,
  child: VitText('Comfortable padding'),
)
```

### Interactive Cards

Make cards interactive with tap, long press, and double tap handlers. Interactive cards display ink splash effects on touch, providing visual feedback to users. This is perfect for navigation cards, selection items, or any tappable content.

```dart
VitCard(
  onTap: () {
    print('Card tapped');
  },
  onLongPress: () {
    print('Card long pressed');
  },
  onDoubleTap: () {
    print('Card double tapped');
  },
  child: VitText('Interactive card'),
)
```

### Loading States

Display skeleton loading states while content is being fetched. The card automatically shows a shimmer effect that preserves the layout, providing excellent user experience during data loading.

```dart
VitCard(
  isLoading: true,
  child: Column(
    children: [
      VitText('Title'),
      VitText('Content'),
    ],
  ),
)
```

You can also use VitLoadingScope to control loading states for multiple cards at once:

```dart
VitLoadingScope(
  loading: isLoadingData,
  child: Column(
    children: [
      VitCard(child: VitText('Card 1')),
      VitCard(child: VitText('Card 2')),
    ],
  ),
)
```

### Accessibility

Add semantic labels for screen readers to improve accessibility. This helps visually impaired users understand the purpose and content of your cards.

```dart
VitCard(
  semanticLabel: 'User profile card for John Doe',
  child: VitText('John Doe'),
)
```

## List Card

VitListCard is a specialized card variant designed for list items. It combines the structure of Flutter's ListTile with card styling, making it perfect for contact lists, settings menus, navigation items, and any list-based interface.

The component provides a consistent layout with leading widgets (icons or avatars), title and subtitle text, and trailing widgets (icons or badges), all wrapped in a card container with full customization options.

### Basic List Card

```dart
VitListCard(
  title: Text('John Doe'),
  subtitle: Text('john@example.com'),
)
```

### List Card with Leading Icon

Leading widgets appear at the start of the card. Use icons, avatars, or custom widgets to provide visual context for the list item.

```dart
VitListCard(
  leading: Icon(Icons.person),
  title: Text('John Doe'),
  subtitle: Text('Software Engineer'),
)
```

### Interactive List Card

List cards support tap and long press interactions, making them perfect for navigation or triggering actions. The entire card becomes a tappable area with visual feedback.

```dart
VitListCard(
  leading: Icon(Icons.settings),
  title: Text('Settings'),
  trailing: Icon(Icons.chevron_right),
  onTap: () {
    Navigator.push(context, SettingsPage());
  },
)
```

### List Card with Avatar

Combine list cards with avatar components for rich contact lists or user directories.

```dart
VitListCard(
  leading: VitAvatar(
    name: 'Jane Smith',
    imageUrl: 'https://example.com/avatar.jpg',
  ),
  title: Text('Jane Smith'),
  subtitle: Text('Online'),
  trailing: Icon(Icons.message, color: Colors.blue),
  onTap: () {},
)
```

### Dense and Three-Line Cards

Control the vertical spacing and text line limits for different layout needs.

```dart
VitListCard(
  dense: true,
  title: Text('Compact list item'),
  subtitle: Text('Uses less vertical space'),
)
```

```dart
VitListCard(
  isThreeLine: true,
  title: Text('Extended content'),
  subtitle: Text('This subtitle can wrap up to three lines of text for displaying longer descriptions'),
)
```

### Selected List Cards

Highlight selected items in lists with selection styling.

```dart
VitListCard(
  selected: true,
  selectedColor: Colors.blue,
  selectedTileColor: Colors.blue.shade50,
  title: Text('Selected item'),
  trailing: Icon(Icons.check),
)
```

### List Card Loading State

List cards also support skeleton loading states, showing placeholder content while data loads.

```dart
VitListCard(
  isLoading: true,
  leading: Icon(Icons.person),
  title: Text('Loading...'),
  subtitle: Text('Please wait'),
)
```

            text: 'Read More',

  ),

## Best Practices

1. **Choose the Right Variant**: Use different card variants to establish visual hierarchy and distinguish between types of content in your interface.

2. **Consider Elevation Carefully**: Higher elevation creates visual hierarchy but use it sparingly. Too many elevated cards can make interfaces feel cluttered.

3. **Use VitListCard for Lists**: When building lists of similar items (contacts, settings, menus), prefer VitListCard over regular VitCard for consistency.

4. **Loading States**: Always implement loading states for cards that display fetched data. This provides better user experience during data loading.

5. **Interactive Feedback**: When cards are tappable, ensure they have appropriate trailing icons (like chevron_right) to indicate interactivity.

6. **Consistent Padding**: Use visual density to maintain consistent padding across your app rather than hardcoding padding values.

7. **Accessibility**: Always provide semantic labels for cards that contain important information or actions.

8. **Group Related Content**: Use cards to group related information together, making your interface easier to scan and understand.
